## v1.10.0 - 2026-04-30
### Changed
* GetGDSSessionData.Create (string) renamed to CreatedAt (time.Time) and Expiry (string) renamed to ExpiresAt (time.Time); custom UnmarshalJSON parses RFC3339 timestamps from the API — callers no longer need to parse timestamps manually
* assessHealth now produces three severity tiers: healthy, warning, and critical. CPU > 95%, memory > 95%, connection usage > 95% (when known), and page cache hit rate < 20% all trigger critical. Previous warning thresholds are unchanged. Critical severity is monotonic — it is never downgraded by a lower-severity condition evaluated afterwards.
* GetSnapshotData.Timestamp changed from string to time.Time; custom UnmarshalJSON parses the RFC3339 string returned by the Aura API — callers no longer need to parse timestamps manually
