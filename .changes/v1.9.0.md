## v1.9.0 - 2026-04-29
### Added
* Added WithInsecureBaseURL option that bypasses the HTTPS enforcement of WithBaseURL — intended for local development and in-process testing (e.g. httptest.Server) only; never use against a real Aura environment
### Changed
* Renamed error_types.go to errors.go — follows Go convention for files holding error type definitions
* Merged client_types.go into client.go — AuraAPIClient struct, config, and functional options are now co-located with NewClient
* Merged {domain}_service.go + {domain}_types.go into single domain files (instances.go, tenants.go, snapshots.go, cmek.go, graphanalytics.go, prometheus.go) — each domain is now self-contained
* Merged test_helpers_test.go and test_helpers_types_test.go into a single mocks_test.go — mock type definitions and their method implementations are now co-located
* Renamed gds_sessions_service.go / gds_sessions_types.go / gds_sessions_test.go to graphanalytics.go / graphanalytics_test.go — name now matches the Aura API path (graph-analytics) and the AuraAPIClient field (GraphAnalytics)
* Renamed internal/httpClient package to internal/httpclient — package name now follows Go's all-lowercase convention; import path updated from internal/httpClient to internal/httpclient across all consumers
* Renamed instance_test.go to instances_test.go for consistency with the new instances.go source file
* Renamed blackbox_test.go to integration_test.go — the name now describes what the tests are (httptest.Server integration tests) rather than how they are structured (external package)
* Simplified internal/api file names: api_service.go → api.go, api_types.go → types.go, api_service_test.go → api_test.go — the _service suffix was redundant given the package is already named api
* Updated .gitignore to remove stale internal/auraClient/coverage.out entry and confirm internal/httpclient/coverage.out is correctly excluded
### Removed
* Deleted all tombstone/no-op files (test_helpers.go, test_helpers_types.go, api_test.go, and the empty _service.go/_types.go stubs) and coverage.out from source control — these files served only as migration markers and are no longer needed
### Fixed
* httpclient: replaced retryablehttp's default CheckRetry policy with a network-only policy that retries solely on connection-level errors — HTTP responses (including 5xx) are now always returned to the caller rather than being treated as retryable failures
