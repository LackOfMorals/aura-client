## v1.3.0 - 2026-01-29
### Added
* get metrics using prometheus endpoints that the instances have. 

## v1.2.1 - 2026-01-28
### Removed
* Removed authMgr.go and requests.go as they are now both deprecated 

## v1.2.0 - 2026-01-22
### Changed
* Architecture changed to high level service - instances, snapshots etc --> API requests -> http client.  Should provide more flexibility

## v1.1.0 - 2026-01-21
### Changed
* The various services that represent each area of the Aura API, instances, snapshots, tenants, etc.. have been abstracted with interfaces. This makes mocking , for tests, easier.  Tests are updated"

## v1.0.6 - 2026-01-21
### Changed
* instances service checks instance id is valid where needed.  Added instance id validation func in utils. fixed typo in tenants.go and in tenants_test.go

## v1.0.5 - 2026-01-21
### Changed
* Modified cmek function to check for tenant id, and if valid, use it as a filter when listing cmeks

## v1.0.4 - 2026-01-04
### Changed
* Remove WithClientID and WithClientSecret and replaced with a single option for credentials, WithCredential

## v1.0.3 - 2026-01-04
### Added
* Added configuration option to set base url and version of the Aura API

## v1.0.2 - 2026-01-04
### Changed
* Switched to use "github.com/hashicorp/go-retryablehttp" from net/http and added retries. Max retry can be set as part of the client configuration

## v1.0.1 - 2026-01-04
### Changed
* This is the first release of a Go package that assists with using Neo4j Aura API.

## v1.0.0 - 2025-11-20
### Added
* Now using Changie for changefile generation
### Changed
* When using a client, removed all mention of Service to cut down the clutter
* Removed exports of all types apart from those needed to create and update an instance.  reduces clutter and tidies up what is exposed.
